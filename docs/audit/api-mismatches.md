### 3.3.4 GET /api/v1/me/articles/{id}/ai — **MISMATCH (no consumer + shape divergence)**

The frontend never calls this endpoint directly. The reader page builds an
`ArticleAiInsights` object via `normalizeArticleAiInsights(article)` from the
fields embedded in `Article.ai_metadata` returned by `/articles/{id}`.

| Backend (`MeArticleAiResponse`) | Frontend (`ArticleAiInsights`) | Match? |
| --- | --- | --- |
| article_id | — | extra |
| tier | — | extra (basic/verified/premium projection key) |
| sentiment | — | not surfaced (frontend reads `ai_metadata.sentiment.label` instead) |
| sentiment_score | — | not surfaced |
| sentiment_rationale | — | not surfaced |
| sentiment_aspect | — | not surfaced |
| summary_one_sentence | summary | semantically same but different field name |
| summary_three_sentences | abstract_text | semantically same but different field name |
| summary_key_points (`Value` of strings) | key_points (`string[]`) | shape mismatch — backend keeps it as a JSON value |
| headline_keywords (`Value`) | keywords (`string[]`) | same shape mismatch |
| tags | tags | yes (both `string[]`) |
| keywords | keywords | yes (both `string[]`) |
| ai_metadata | (used as the actual source of truth) | yes |
| — | risk_score, risk_level, risk_dimensions, recommendations, entities | only present in `ai_metadata`, not on the dedicated endpoint |

**Decision needed**: either (a) frontend migrates to call `/me/articles/{id}/ai`
and adopt that DTO, or (b) backend retires the per-article AI route. Today both
paths coexist but only the embedded `ai_metadata` is consumed.

## Recommended follow-up tasks

The lead can decide whether each row warrants its own task; this audit does not
self-author them.

1. **AI governance backend** — implement the `/api/v1/admin/ai/*` policy /
   prompt / metrics / experiments endpoints the SPA already depends on
   (§3.2 bucket 1).
2. **Article enrichment route consolidation** — pick a single source of truth
   between embedded `ai_metadata` and `/me/articles/{id}/ai`, retire the loser
   (§3.3.4).
3. **Reader-page enrichment hooks migration** — `use-article-summary`,
   `use-sentiment`, `use-kg-extract` all 404; either swap them onto
   `/me/articles/{id}/ai` or implement the missing per-article enrichment
   verbs (§3.2 bucket 2).
4. **Super-tenant drawer extras** — wire the F.7/F.8 backend (`users`,
   `suspend`, `admin/reset-password`, `export`, `exports`, `exports/:id`)
   into `use-admin-tenants` (§3.1 bucket 1).
5. **ReBAC admin pages** — `/admin/authz/relations` list, `/admin/roles`,
   `/admin/permissions` are referenced from admin matrices but unimplemented
   (§3.2 bucket 3).
6. **Pin path drift** — `use-pins.ts` calls `/articles/pins`; backend serves
   `/article-pins`. Either fix the hook or add an alias route (§3.2).
7. **Feedback PUT vs PATCH** — `use-feedback.ts:62` issues PUT, backend exposes
   only PATCH (§3.2).
