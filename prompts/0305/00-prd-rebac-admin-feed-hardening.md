# 0305 PRD：ReBAC、管理面板、阅读体验与真实资讯链路收口

## 背景

LawSaw 在本轮开发前已经具备多租户隔离、基础 RBAC、真实抓取、知识图谱、报告与 AI 服务等能力，但存在以下阻断交付的问题：

- 全站骨架层动效不稳定，品牌区和侧栏在切页、刷新时出现抖动与错位。
- 右上角 bell 行为错误，点击后跳转设置页，而不是打开消息面板。
- 管理员工作台与用户工作台虽已设计，但缺少完整的真实运行态闭环。
- ReBAC 在数据模型、查询裁剪与页面消费层未形成统一的、可解释的落地实现。
- 公网真实 source 可用性不足，影响分级阅读、运营位、知识图谱与 RAG 的完整验收。
- 文档交付存在编码污染与状态漂移，无法作为企业级规范化产物直接交付。

## 产品目标

本轮工作的目标不是“补几个点”，而是把系统从“已有功能雏形”推进到“可交付、可验证、可持续演进”的状态。

核心目标如下：

1. 稳定全站骨架层交互，消除品牌区与侧栏布局抖动。
2. 将 bell 改造成真实消息中心入口，而非设置页跳转。
3. 基于现有租户隔离设计，把 ReBAC、频道、Banner、置顶、Feed、来源可见性、管理员面板与用户面板真正串成闭环。
4. 以真实公网 source、真实数据库、真实队列、真实浏览器验收为准，不使用 mock。
5. 把 `prompts/0305` 沉淀成当前轮唯一可信的 canonical 交付文档集。

## 目标用户与角色

### 平台与租户治理角色

- `super_admin`
  - 跨租户治理、系统级配置、渠道源策略、AI 治理与审计。
- `tenant_admin`
  - 租户内用户管理、关系元组治理、频道、Banner、置顶、反馈与运营配置。

### 终端用户角色

- `basic_user`
  - 查看受限频道资讯，不暴露完整来源元数据。
- `verified_user`
  - 查看扩展频道与更多阅读上下文。
- `premium_user`
  - 查看完整来源与更完整分析能力。

## 本轮范围

### P0：必须收口

- UI 骨架稳定性
- bell 消息面板
- 公网真实 source 至少再打通并在当前租户验收
- `/me/feed` 聚合与服务端裁剪
- Banner 生命周期运行态
- 置顶真实接口与 feed 展示
- RAG 真正落地到当前租户
- 浏览器级稳定回归

### P1：高价值增强

- 管理员工作台入口完善
- 用户面板入口完善
- 频道 / Banner / 置顶的运营闭环
- 文章详情来源分级视图

### P2：AI 与知识图谱深化

- SiliconFlow chat / embedding / rerank 分层接线
- semantic / ask 真实返回
- AI telemetry 回读
- embedding 维度归一化与失败降级策略

## 非目标

以下不在本轮作为主要目标推进：

- 大规模 UI 重构为全新设计系统
- 引入新的浏览器渲染中间件镜像
- 修改已应用的历史迁移
- 清理仓库中所有历史实验文件与旧轮次产物

## 强约束

- 禁止 mock、模拟数据、假链路。
- 只做最小必要容器重建，不误杀无关进程。
- 不额外下载 Playwright，不引入新的重型镜像。
- 所有结论以真实 API、真实数据库、真实 worker 日志、真实浏览器验收为准。
- 所有图标禁止使用 Emoji，统一采用现有图标库或非 Emoji 标识字符串。

## 验收标准

本轮交付满足以下条件视为通过：

- `/zh/me/feed` 在当前运行态展示真实频道、横幅、置顶与文章列表。
- 右上角 bell 点击后打开消息中心面板。
- 当前轮新租户下，`miit_gov` 与 `samr_gov` 均完成真实抓取入库。
- 当前轮新租户下，RAG 已产生真实 `article_chunks` 与 `embedding`，并能返回 semantic / ask 结果。
- 管理员关键页面可在浏览器中打开并显示真实数据。
- `prompts/0305` 文档集为干净 UTF-8，可作为本轮交付的唯一规范说明。
